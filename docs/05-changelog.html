版本历史

开发

  • 添加了需要 retina 分辨率的新栅格切片
  • 对 MapImageFactory 添加了mipmap，实时压缩（通过 DXT 压缩）以及 retina 分辨率支持
  • 建筑平顶（地形顶部）现在可以被正确渲染
  • 复杂的楼宇数据现在可以被正确渲染（删掉孔洞、浮动部件等）
  • PoiGeneration 示例加入了 3D 空间游戏对象放置功能（gameobject placement）

v0.5.1

2017年5月1日

  • 可重新设定地形高度（修复了超出范围的异常情况）
  • 修复了 MeshFactories 未能正确序列化的问题
  • 修复了在创建新的 MeshFactory 时，空（null）参数的异常情况

v0.5.0

2017年4月26日

  • 增加了对UWP的支持
    • 与我们分享您的 Hololens 创作吧！
  • 修复了切片转换的精度问题
    • 用 Vector2d 替换了Geocoordinate
  • Mapbox API 令牌现在存储在 MapboxAccess.txt 中
    • MapboxConvenience已被删除
  • 添加了LocationProviders和示例场景，以根据纬度/经度构建地图或放置对象
  • Mesh Generation（网格生成）：
    • 一般性能改进（局部切片几何图形）
    • Map factories 的定制编辑器
    • 添加了新的 MergedModifierStack，它将减少转换次数并在密集地图中调用渲染（Draw Call）（即准备数据并通知程序渲染）
    • 建筑外立面的连续 UV
    • DirectionsFactory 现在用于绘制完整的几何图形，而不仅仅是航点（waypoints）
    • 修复 PolygonMeshModifier.cs 中偶尔出现顶点不匹配的情况（导致索引超出范围）

v0.4.0

  • 将 mapbox-sdk-unity-core 更新到 v1.0.0-alpha13；要素矢量切片超出缩放范围
    • README.MD 中归属信息（attribution）指南的更新
    • 添加了Conversions.cs 和 VectorExtensions.cs，以实现从地理坐标到 unity 坐标空间的简单转换

v0.3.0

  • 为网格生成（mesh generation）增加了新的的基础结构
    • 为生成基本的、添加了样式的目标点矢量网格添加了新的演示程序
    • 为迅速实现矢量切片+地形（动态切片加载）添加演示程序
    • 为 Mapbox Directions＆Traffic 添加了新的演示程序
    • 已弃用的演示程序
    • 已弃用的方向组件演示程序

v0.2.0

  • 对 Mapbox 样式增加了核心 sdk 支持
    • 就速度和延迟解码（lazy decoding）进行了矢量切片解码优化
    • 添加了归属信息预设
    • 新 Directions 示例
    • 所有更新的示例脚本都得到了改进，以使用 MapboxConvenience 对象

v0.1.1

  • 从 link.xml 中删除了孤立的参数，这之前导致版本（build）出错
    • 将 JSON 实用程序移动到 Mapbox 命名空间，以避免与先前的框架产生冲突
  • 改进此文档
